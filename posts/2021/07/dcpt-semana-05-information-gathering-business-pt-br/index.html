. Lab 01

Utilizando o Hunter

Faça cadastro no site Hunter para conseguir visualizar os resultados.

Depois basta digitar o domínio no campo de busca:

Utilizando o TheHarvester

Caso ainda não tenha instalado, instale o theHarvester:

apt install theharvester

Fazendo a busca:

theHarvester -d businesscorp.com.br -l 100 -b google

Flags

-d = pesquisa por domínio

-l = limite de busca

-b = indica o local da busca, nesse caso indicamos o google.

Para outros “sources” de busca digite theHarvester -h, algumas fontes de dados precisam de configuração de api. Essa configuração de api deve ser feita em /etc/theHarvester/api-keys.yaml

Colete os emails encontrados.

. Lab 02

Utilizando os operadores do google hacking, vamos buscar a key no domínio.

Operadores google hacking:

site:= busca no site especificiado

inurl:=busca na url

intitle:=busca no título

intext = busca no texto

filetype= busca por tipo de arquivo

ext= busca por extensão de arquivo

cache= busca no cache

Tente buscar por arquivos txt,php,doc e outras extensões. Além de buscar por arquivos com nomes específicos como: pass, senha, config, configuracao, admin etc.

Vamos iniciar a busca utilizando os operadores sitee o opertador extpara buscar por arquivos de texto no domínio.

O google retornou 1 resultado, ao abrir o site temos uma lista de diretórios configurados com “Dissalow” ou seja essas páginas não são indexadas pelo google, nossa key pode estar em um deles.

Infelizmente ao consultar todos os caminhos, ainda não encontramos a key.

. Lab 07

Agora precisamos analisar os metadados do arquivo encontrado no lab anterior. exiftool. Execute:

exiftool NOME_DO_ARQUIVO

Verifique os dados retornados para encontrar o nome do usuário.

Além dessa forma, podemos utilizar o script que foi demostrado em aula. O que esse script faz é exatamente uma busca em um site por um tipo de arquivo, utilizando os operadores do google.

Exemplo:

.\metafind.sh businesscorp.com.br txt

Resultado da busca:

Script .\metafind.sh

  // #!/bin/bash
#Metafind search, download based on Google Dorcking
#

export RED='\033[0;31m'
export GREEN='\u001b[32;1m' 
export NC='\033[0m'
export Y='\u001b[33m' 

if [ "$1" == "" ] & [ "$2" == "" ]
then
	echo -e "${RED}================================================="
	echo -e " Sintax: ./metafind www.site.com filextention"
	echo -e "Example ./metafind www.tesla.com.br pdf"
	echo -e "================================================="${NC}
else
	
	echo -e "${GREEN}Start search..."${NC}
#Simple loading status
	echo -ne ${GREEN}'#####                     (33%)\r'

	sleep 1

	echo -ne '#############             (66%)\r'

	sleep 1

	echo -ne '#######################   (100%)\r'${NC}

	echo -ne '\n'
# End of loading status


	lynx --dump "https://google.com/search?&q=site:$1+ext:$2" | grep ".$2" | cut -d "=" -f2 | egrep -v "site|google" | sed 's/...$//' > temp 
      	                           
echo -e "${Y}=================================================="
echo -e "=============File URL =============================="
echo -e "======================================================"${NC}
cat temp 

fi                        


for url in $(cat temp);do wget -q $url;done
echo -e "${GREEN}Download files"

echo -ne '#####                     (33%)\r'
sleep 1
echo -ne '#############             (66%)\r'
sleep 1
echo -ne '#######################   (100%)\r'${NC}
echo -ne '\n'

echo -e "${RED}################################################"
echo -e " Complete File INFO:"
echo -e "#################################################"${NC}

for arquivo in $(ls | grep .$2);do exiftool $arquivo;done
