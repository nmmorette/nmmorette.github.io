Lab 03

Para esse lab esxiste mais de uma forma de encontrar a resposta.

DNS transfer

Vamos fazer uma consulta utilizando o comando host:

 host -t ns businesscorp.com.br

Com o resultado do comando teremos os nameservers do domínio, assim podemos rodar host e indicar o endereço para consulta:

 host -l businesscorp.com.br ENDEREÇO_DO_NAME_SERVER

Em caso de falha, podemos tentar em outro nameserver para verificar.

Brute Force DNS

Caso o método acima não funcione, podemos executar um scritp para tentar descobrir os nomes dos subdomínios.

O script faz um “brute force” de DNS, ele verifica se os subdomínios existem de acordo com uma wordlist. Dependendo do tamanho da sua wordlist a execução do script pode ser demorada.

Eu utilizei a wordlist que criei baseada em wordlists do dirb com outras adições minhas e de outras wordlists que encontrei na internet. subdomain.txt

 ./dnsrecon.sh businesscorp.com.br

Utilize uma das formas descritas acima para encontrar a resposta desse lab.

Resposta infrasecreta.businesscorp.com.br

Lab 07

Para esse lab utilizaremos o script dnstransfer.sh O script verifica se é possível fazer uma transferência de DNS nos nameservers. Se for possível teremos uma cópia dos registros DNS do servidor, isso ocorre quando o servidor DNS não foi configurado corretamente para somente aceitar solicitações de transferências de servidores autorizados.

#!/bin/bash

#verifica entradas do DNS e tenta um ataque de transfer dns para encontrar subdominios
for server in $(host -t ns $1 | cut -d " " -f 4 );do
	host -l -a $1 $server
done
Resposta 0989201883299
